# 8일차: 카드 더미·셔플·드로우 시스템 구현

## 개발 목표

전투용 카드 인스턴스를 드로우 더미, 손패, 버린 카드 더미, 제외 더미로 관리하고 전투 시작 및 턴 진행에 필요한 셔플과 드로우 구조를 구현한다.

## 구현 내용

### 1. 카드 더미 구조 구현

- 드로우 더미 구현
- 손패 구현
- 버린 카드 더미 구현
- 제외 카드 더미 구현
- 카드 더미 종류를 관리하는 CardPileType 추가
- 카드가 하나의 더미에만 존재하도록 이동 구조 구성

### 2. 카드 이동 기능 구현

- 드로우 더미에서 손패로 이동
- 손패에서 버린 카드 더미로 이동
- 카드를 제외 카드 더미로 이동
- 카드 이동 전후 Instance ID 유지
- 전체 전투 카드 등록부와 카드 위치 분리

### 3. 카드 셔플 구현

- Fisher–Yates 방식의 무작위 셔플 구현
- 전투 시작 시 드로우 더미 자동 셔플
- 버린 카드 재사용 시 드로우 더미 재셔플
- 셔플 과정에서 카드 인스턴스 정보 유지

### 4. 카드 드로우 구현

- 전투 시작 시 카드 8장 드로우
- 턴 시작 시 카드 4장 드로우
- 실제로 드로우한 카드 수 반환
- 드로우 가능한 카드가 부족한 상황 처리
- 빈 더미 접근 예외 방지

### 5. 버린 카드 재사용 구현

- 드로우 더미 소진 여부 확인
- 버린 카드를 드로우 더미로 이동
- 이동된 카드를 무작위로 재셔플
- 제외 카드는 재사용 대상에서 제외

### 6. 손패 정리 구현

- 손패 전체를 버린 카드 더미로 이동
- 턴 종료 후 손패 정리에 사용할 공개 메서드 구성
- Inspector 테스트 메뉴 구성

### 7. 테스트 덱 확장

- 테스트 덱을 4장에서 12장으로 확장
- 시작 손패 8장 검증
- 턴 드로우 4장 검증
- 동일 원본 카드의 독립 인스턴스 생성 확인

### 8. 카드 더미 무결성 검증

- 네 카드 더미의 카드 수 합계 확인
- 카드 중복 배치 확인
- 카드 위치 누락 확인
- Instance ID 유지 확인
- 제외 카드 재사용 방지 확인
- Console 오류 없이 빈 더미 처리 확인

## 완료 결과

전투 카드가 드로우 더미, 손패, 버린 카드 더미, 제외 더미 사이를 안전하게 이동하는 구조를 완성했다. 전투 시작 시 덱을 섞고 8장을 드로우하며, 턴 시작 시 4장을 드로우할 수 있다. 드로우 더미가 소진되면 버린 카드를 다시 섞어 사용하고 제외 카드는 재사용하지 않도록 처리했다.
